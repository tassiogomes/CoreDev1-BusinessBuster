Table Extension
________________________________________________________________________________________________________________	

        • Uso
                - Criar novos documentos (por exemplo, um novo pedido de venda).
                - Editar documentos existentes.
                - Visualizar detalhes dos documentos.
                - Gerenciar linhas associadas ao documento principal (por exemplo, linhas de itens em um pedido de venda).

Propriedades
________________________________________________________________________________________________________________	


        • Propriedades
                - SourceTable: A tabela principal associada à página (por exemplo, Sales Header para pedidos de venda).
                - PageType: Deve ser definida como Document.
                - ApplicationArea: Define a área de aplicação da página (por exemplo, All, Basic, Suite).
                - UsageCategory: Define onde a página será exibida no cliente (por exemplo, Lists, Tasks).


        • Explicação do Snippet
                • PageType = Document: Define que a página é um documento.
                • SourceTable = "Sales Header": Especifica que a tabela fonte é a Sales Header, que contém os dados do cabeçalho do pedido de venda.
                • ApplicationArea = All: Indica que esta página está disponível em todas as áreas de aplicação.
                • layout: Define o layout da página, incluindo campos do cabeçalho e linhas do pedido.
                • actions: Define ações disponíveis na página, como liberar o pedido.


Guia
________________________________________________________________________________________________________________	




		• Criar a Tabela de Cabeçalho (Header)
			- Propósito: Armazena as informações principais do documento (ex.: número do documento, descrição).
			- Chave Primária: Defina um campo como chave primária (geralmente No. ou Document No.).

		• Criar a Tabela de Linhas (Lines)
			- Propósito: Armazena os detalhes do documento (ex.: itens, quantidades).
			- Chave Primária: Defina uma chave primária composta por Document No. (referenciando o cabeçalho) e Line No..

		• Criar a Página de Documento (Document Page)
			- Fonte: Use a tabela de cabeçalho como a tabela fonte.
			- Layout:
			  - Grupo Geral: Adicione os campos principais do cabeçalho.
			  - Grupo de Linhas: Adicione uma parte para exibir as linhas do documento.
			  - SubPageLink: Conecte as linhas do documento ao cabeçalho através de Document No. = No..

		• Criar a Página de Linhas (ListPart)
			- Fonte: Use a tabela de linhas como a tabela fonte.
			- Layout:
			  - Repeater: Defina um repetidor para listar os registros das linhas do documento.
			  - Campos: Adicione os campos que precisam ser exibidos nas linhas (ex.: Item No., Quantity).

		Ordem das Etapas:
			- Tabela de Cabeçalho (Header): Define os campos principais do documento.
			- Tabela de Linhas (Lines): Define os detalhes do documento e relaciona cada linha ao cabeçalho através do Document No..
			- Página de Documento (Document Page): Configura a exibição do cabeçalho e inclui uma parte para exibir as linhas.
			- Página de Linhas (ListPart): Configura a exibição dos detalhes do documento na forma de uma lista.

		Relações Importantes:
			- SubPageLink: Na Document Page, use o SubPageLink para conectar as linhas do documento ao cabeçalho (Document No. = No.). (campo da tabela linhas = campo da tabela header)
			- Repeater: Na ListPart, use um repetidor para exibir os registros das linhas.





Notas
________________________________________________________________________________________________________________	

	• Criar e Editar Documentos: 
		
		- Na página de documento, você cria e edita documentos como pedidos de venda. Cada linha do documento pode corresponder a uma `Journal Line` que registra a transação financeira.
	
	• Journal Lines: 
		
		- As Journal Lines são linhas individuais de transações financeiras registradas em diários. Por exemplo, em um pedido de venda, as Journal Lines podem representar cada item vendido, cada pagamento recebido, etc.
	
	• Registro no Ledger:
		
		- Quando um documento é salvo ou postado na página de documento, suas transações são registradas no `Ledger`. Cada `Journal Line` é registrada no `Ledger` conforme definido pelas configurações contábeis.

	• Registers (Registros):

		- Registers referem-se aos registros ou entradas específicas em um diário contábil ou outra forma de registro financeiro. Por exemplo, um registro de venda em um diário de vendas.
	
	• Posted Documents:
	
		- Após o documento ser contabilizado (`Posted`), ele se torna um `Posted Document`, refletindo as transações efetivamente registradas e afetando os saldos das contas no `Ledger`.


Sales Order Header: Esta tabela representa o cabeçalho da ordem de vendas, com campos como No. (número da ordem) e Customer No. (número do cliente).
Sales Order Line: Esta tabela representa as linhas da ordem de vendas, com campos como Line No. (número da linha), Document No. (número do documento) e Item No. (número do item).
AutoSplitKey: Na tabela Sales Order Line, a propriedade AutoSplitKey foi adicionada para especificar um campo que será usado para dividir automaticamente as entradas de linha em subgrupos. Isso é útil quando você precisa categorizar automaticamente as linhas da ordem de vendas com base em algum critério específico.
