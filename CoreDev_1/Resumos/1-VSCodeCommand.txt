		• App.json - é a sua app enquanto o launch é a sua configuração, não tem nada a ver.
		• idRange - número que você cria uma extensão tem que estar nesse range.
		• CustomerList.PageExt.al - objectname.objecttype.al padrão para nomear o arquivo e o nome do objeto.
		• Um file pode ter vários objetos, mas o ideal é ter só um por file.	


		• ttable - comando para criar uma tabela nova, renomear como acima.
		• caption - ao criar um field novo, tem que usar o caption, que é o que será mostrado na user interface.
		• DataClassification - é usado para efeitos de proteção e RGPD, os comuns são:
			- ToBeClassified: Indica que o dado ainda precisa ser classificado.
			- CustomerContent: Dados inseridos pelo cliente.
			- SystemMetadata: Dados de sistema, metadados.
			- EndUserIdentifiableInformation (EUII): Informações que podem identificar diretamente um usuário final.
			- AccountData: Dados relacionados a contas.
			- Confidential: Dados confidenciais que requerem proteção especial.

		• Primary Key - usar um dos campos introduzidos nos fields nos campos Keys e colocar como Clustered = true

        			key(PK; "No.")

            				Clustered = true;
        				}
